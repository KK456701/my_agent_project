# Python 架构最佳实践

## 单一职责原则 (SRP)
- 一个模块/类只做一件事
- 反模式: 认证 + 日志 + 限流全在一个函数里

## 依赖注入
- 不在函数内 `get_db()` 硬依赖
- 改为参数注入: `def login(db: Connection, ...)`

## 配置管理
- 魔法数字 → 模块级常量 `TAX_RATE = 0.15`
- 硬编码配置 → 环境变量 / 配置文件

## 错误处理
- 不用 `print()` 做日志 → `logging` 模块
- 不用裸 `except:` → 指定异常类型
- 异常安全: `finally` 释放资源

## 可测试性
- 纯函数优先（无副作用）
- 数据库/网络调用可 mock
- 避免全局状态

## Python 特有
- 类型注解: `def func(x: int) -> str:`
- 上下文管理器: `with` 管理资源
- 数据类: `@dataclass` 替代字典
